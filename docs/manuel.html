Version 1.0

Projeter une vidéo, exactement là où il faut.

Toolbox transforme un ordinateur — ou un Raspberry Pi — en node de projection : lecture vidéo, mapping 4 coins, correction couleur, le tout piloté depuis n'importe quel navigateur, en OSC ou en MIDI. Il redémarre seul, en plein show, après une coupure de courant.

Prise en main

Démarrage rapide

Cinq minutes entre le téléchargement et une vidéo mappée sur le mur.

  1. Dézippe le pack et lance toolbox-node.exe La fenêtre de sortie s'ouvre (noire : c'est normal, rien n'est encore projeté).
  2. Ouvre http://localhost:8080 Depuis le même ordinateur, ou http://IP-de-la-machine:8080 depuis un téléphone sur le même réseau.
  3. Onglet Médias : dépose une vidéo, puis « Charger » Pour un gros fichier, copie-le directement dans le dossier media/ à côté du logiciel.
  4. Lecture ▶ La vidéo apparaît dans la fenêtre de sortie, avec le son. Le badge « img/s » indique la fluidité réelle.
  5. Onglet Mapping : tire les 4 coins L'image suit en direct. F11 dans la fenêtre de sortie pour le plein écran.
Prise en main

Installation

Les binaires se téléchargent depuis la page Releases du dépôt GitHub (aucun compte requis). Un dossier suffit : tout est portable, les dossiers media/ presets/ logs/ se créent à côté au premier lancement.

Windows — pack complet

toolbox-node-windows-x64-gstreamer

Lecture vidéo incluse (décodeurs GStreamer livrés dans le zip). Dézipper, double-cliquer, rien à installer. Recommandé.

Windows — léger

toolbox-node-windows-x64

Sans lecture vidéo (mires et calibrage uniquement), ~10 Mo. Pour piloter ou tester sans projeter.

Ubuntu / Debian

toolbox-node-linux-x64

Vidéo : sudo apt install gstreamer1.0-plugins-base gstreamer1.0-plugins-good gstreamer1.0-plugins-bad gstreamer1.0-libav

Raspberry Pi 4 / 5

toolbox-node-raspberrypi-arm64

Mêmes paquets GStreamer que Ubuntu. L'installeur deploy/install.sh configure le service systemd (démarrage au boot, relance en cas de crash).

Prise en main

L'interface web

Tout se pilote depuis le navigateur — ordinateur, tablette ou téléphone. Plusieurs appareils peuvent être connectés en même temps : chacun voit les changements des autres en direct.

Dashboard

Transport (lecture, pause, stop), barre de progression, volume, playlist, badge img/s — et l'aperçu de la sortie : ce que projette le node, rafraîchi en continu.

Mapping

Les 4 coins (souris, tactile ou flèches du clavier), rotation, miroirs, recadrage, mires de test, mappings enregistrés, choix de l'écran de sortie.

Couleur

Luminosité, contraste, gamma, saturation, teinte et gains RVB. Double-clic sur un curseur : retour au neutre.

Médias

Dépôt de fichiers, renommage, suppression. Les fichiers copiés directement dans media/ apparaissent aussi.

Presets

L'état complet du node (média, mapping, couleur, playlist) sauvegardé sous un nom, rechargeable en un clic — ou en fondu (durée réglable) : coins, couleur, effets et volume glissent vers la scène cible sans interrompre la lecture.

Fonctions

Un interrupteur par fonction du node (OSC, MIDI, lecteur, parc réseau, fondus, aperçu…). Désactivée = réellement arrêtée, zéro ressource ; réactivée = redémarrage immédiat, sans relancer le node. Mémorisé dans fonctions.json.

Logs & Système

Journal en direct du node (et sur disque : un fichier par jour dans logs/, gardé 14 jours — lisible même après un crash), charge machine, mémoire, température (Pi).

Sur le terrain

Calibrer un vidéoprojecteur

  1. Choisis la sortie Onglet Mapping, carte « Sortie » : sélectionne l'écran du projecteur et coche « Plein écran ». Appliqué immédiatement, retenu pour les prochains démarrages.
  2. Affiche une mire « Grille » pour la géométrie, « Coins » pour identifier chaque angle (0 = haut gauche, 1 = haut droit, 2 = bas droit, 3 = bas gauche).
  3. Cale les 4 coins sur la surface Tire les poignées jusqu'à ce que la grille épouse le mur, l'écran ou le décor. Clavier : flèches = pas de 0,001, Maj + flèches = 0,01. Les coins peuvent sortir du cadre (marge de 50 %).
  4. Ajuste orientation et recadrage Rotation par quarts de tour, miroirs H/V, recadrage par bord (projecteur au plafond, image à rogner…).
  5. Enregistre le mapping Carte « Mappings enregistrés » : donne-lui un nom. Tu pourras le recharger sans toucher au média ni à la couleur — même en pleine lecture.
AstuceLa case « Mapping actif » bypasse toute la déformation sans perdre les réglages : pratique pour comparer avant/après ou projeter plein cadre temporairement.
Sur le terrain

Lire des vidéos

La lecture s'appuie sur GStreamer : MP4 (H.264/H.265), WebM, MOV, MKV et la plupart des formats courants, avec accélération matérielle quand la machine en a une. Le badge img/s à côté de la barre de lecture affiche le débit d'images réellement projetées — s'il colle à la cadence du fichier (24, 25, 30…), tout va bien.

Boucles et playlists

  • Boucle « un » : le média repart sans coupure (enchaînement natif GStreamer, aucun trou noir).
  • Boucle « tout » : enchaîne la playlist et recommence.
  • Playlist : construite dans le Dashboard ; suivant/précédent au clic, en OSC ou en MIDI.

Sources externes

Le node ne lit pas que des fichiers — champ « Source externe » de l'onglet Médias (ou /load en OSC) :

  • capture://0 — première webcam ou carte d'acquisition HDMI/USB (le boîtier déformeur entre une source et le projecteur) ;
  • rtsp://…, srt://…, http(s)://… — flux réseau ;
  • ndi://Nom de la source — flux NDI (plugin GStreamer + runtime NDI requis, message clair sinon) ;
  • une image fixe (PNG, JPEG…) s'affiche en continu.

Les sources live n'ont ni durée, ni barre de progression, ni seek.

Effets

Onglet Couleur, carte « Effets » : pixellisation, postérisation, bruit animé, netteté, miroir kaléidoscope. Une intensité 0..1 par effet (0 = inactif) — un fader MIDI ou OSC (/effect/pixelate…) suffit à l'activer et le doser. Calculés par le GPU comme le reste.

Gros fichiers

Le dépôt via le navigateur est plafonné (2 Go par défaut, réglable par [limits] max_upload_mb). Au-delà, copie simplement le fichier dans le dossier media/ : il apparaît dans l'onglet Médias.

Priorité d'affichageUne mire de test sélectionnée passe toujours devant la vidéo (le calibrage prime). Pense à remettre la mire sur « Off » pour revoir l'image.
Sur le terrain

Show autonome

Objectif : brancher la machine, elle projette. Trois briques :

  1. Un preset de show Règle tout (média, mapping, couleur), puis onglet Presets → sauvegarde sous un nom, par exemple show.
  2. Le mode kiosque dans node.toml [startup] preset = "show" et autoplay = true : à chaque démarrage du node, le preset se charge et la lecture démarre.
  3. Le lancement au démarrage de la machine Windows : copie install-autostart-windows.bat à côté de l'exe et double-clique (retrait : relancer avec --remove). Pi/Linux : le service systemd installé par deploy/install.sh démarre au boot et relance en cas de crash.

Contrôle OSC

Auto-découverte OSCQuery (Chataigne)

Dans Chataigne, ajoute un module OSCQuery : le node s'annonce sur le réseau (mDNS _oscjson._tcp) et apparaît dans la liste — sinon, IP du node et port 8081. Tous les paramètres arrivent avec leurs curseurs bornés et leurs valeurs courantes.

Retour d'état (feedback)

Avec [osc] feedback = "10.0.0.5:9001" dans node.toml, le node renvoie chaque changement en OSC à cette adresse, avec les mêmes adresses que les commandes (/volume, /corner/2, /pattern…). Les curseurs de Chataigne suivent donc le node, même quand le changement vient de la web UI ou du MIDI.

Fichiers du parc

Onglet Médias → carte « Fichiers du parc » : chaque node découvert y apparaît — voir ses médias, lui envoyer le fichier sélectionné, ou « Envoyer à tout le parc » pour copier un média sur toutes les machines d'un coup (flux direct de node à node, rapport par machine).

Demander de l'aide : l'export diagnostic

Système → « Exporter le diagnostic (ZIP) » (ou GET /api/diagnostic.zip) : une archive avec l'état complet, le journal, les infos système, les écrans, les médias et les presets — tout ce qu'il faut pour comprendre un node à distance, sans aucun secret dedans.

Plusieurs projecteurs ensemble

Chaque node s'annonce sur le réseau (page Système → Réseau : liste du parc, lien « Ouvrir », bouton « Identifier » qui flashe la mire coins sur la sortie du node visé). Pour un départ synchronisé : charge un média sur chaque node, envoie /sync/arm à tous, puis /sync/startAt avec la même heure — chaque node part précisément à l'échéance sur son horloge (NTP conseillé).

Verrouillage à la frame (sync niveau 2)

Pour rester calés en continu : un node maître ([sync] role = "maitre") et des suiveurs (role = "suiveur" + maitre = "ip:9010"). Les suiveurs chargent le média du maître, suivent lecture/pause et corrigent leur dérive en douceur (vitesse ±3 %, invisible) — resync dur seulement au-delà de tolerance_ms. Mesuré : dérive sous 2 ms, très en dessous d'une frame. Horloges des machines en NTP obligatoire.

La même grammaire existe en JSON sur POST /api/command et sur le WebSocket /ws — voir la table dans crates/core/src/command.rs.

Contrôle MIDI

Les bindings se déclarent dans node.toml : une note déclenche une commande, un contrôleur continu (CC) pilote un paramètre mis à l'échelle.

[modules]
midi = true

[midi]
port = "APC"                 # sous-chaîne du nom du port (absent = premier port)

[[midi.bindings]]
note = 60                    # C4 déclenche la lecture
command = { cmd = "play" }

[[midi.bindings]]
cc = 7                       # le fader 7 pilote le volume
scale = "volume"             # 0..127 → 0..1 (aussi : brightness, gamma, hue…)

[[midi.bindings]]
note = 62
channel = 10                 # canal précis (1..16)
command = { cmd = "preset_load", name = "scene_01" }

Configuration — node.toml

Un fichier optionnel à côté du binaire. Tout champ absent prend sa valeur par défaut : un node.toml vide (ou manquant) démarre en mode portable. Le fichier complet documenté : node.toml.example.

[startup]Mode kiosque : preset + autoplaydésactivé
[paths]Dossiers média, presets, logsrelatifs au binaire

L'écran et le plein écran choisis dans l'UI sont mémorisés dans sortie.json (créé à côté de node.toml) et priment sur [output] au démarrage suivant. Supprime ce fichier pour revenir à la config.

Dépannage

SymptômeCause probableSolution
Fenêtre de sortie noireRien à projeter Normal sans mire ni lecture en cours. Choisis une mire (onglet Mapping) ou lance un média.
La vidéo ne se lance pasBinaire léger ou GStreamer absent Prends le pack …-gstreamer (Windows) ou installe les paquets GStreamer (Linux/Pi). Les logs indiquent « backend simulé » dans ce cas.
Image saccadéeRendu processeur actif Vérifie « rendu GPU actif » dans la page Logs. Sinon : pilotes graphiques (Vulkan) à mettre à jour ; le badge img/s permet de comparer.
UI inaccessible depuis le téléphonePare-feu ou mauvaise IP Autorise le port 8080 (privé) sur la machine ; vérifie que les deux appareils partagent le même réseau.
Écran du projecteur introuvableBranché après le lancement Change d'écran dans la carte « Sortie » : la liste est re-détectée à chaque application.
Le fichier refuse de s'uploaderPlus gros que la limite Copie-le directement dans media/, ou augmente [limits] max_upload_mb.
Un preset ne charge pasFichier modifié à la main Les presets invalides sont refusés (valeurs hors bornes). Le journal donne la raison exacte.

Chaque incident laisse une trace dans l'onglet Logs — c'est le premier endroit où regarder. En cas d'échec de la CI du projet, les journaux complets sont publiés sur les branches ci-logs-* du dépôt.
